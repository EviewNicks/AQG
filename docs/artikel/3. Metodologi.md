# 3. Metodologi

Bagian ini memaparkan kerangka kerja penelitian yang sistematis dalam pengembangan sistem *Automatic Question Generation and Distractor Generation* (AQG-DG) berbasis IndoT5. Pendekatan yang diusulkan mengintegrasikan teknik *Parameter-Efficient Fine-Tuning* (PEFT) melalui *Low-Rank Adaptation* (LoRA) untuk mengoptimalkan model pada domain pendidikan pemrograman Python berbahasa Indonesia. Alur kerja penelitian ini dirancang secara modular untuk memastikan setiap tahapan—mulai dari persiapan data hingga evaluasi—memiliki landasan teoretis dan teknis yang kuat, sebagaimana divisualisasikan dalam Gambar 1.

Pemilihan arsitektur *encoder-decoder* T5 didasarkan pada efektivitas kerangka kerja *text-to-text* yang menyatukan berbagai tugas NLP ke dalam format tunggal [1]. Dalam konteks penelitian ini, arsitektur tersebut memungkinkan model untuk melakukan *joint generation* antara pertanyaan dan pengecoh dalam satu proses inferensi, yang terbukti lebih konsisten secara semantik dibandingkan pendekatan *pipeline* tradisional [6].

## 3.1. Akuisisi & Deskripsi Dataset

Persiapan dataset merupakan tahap paling krusial dalam menentukan kualitas luaran model generatif. Penelitian ini mengadopsi strategi **hibrida dua tahap** (*two-stage fine-tuning*) untuk menjembatani kesenjangan antara pengetahuan umum model bahasa dan spesifisitas domain pendidikan pemrograman Python [2]. Strategi ini dirancang untuk memitigasi risiko halusinasi dengan memastikan model tetap *grounded* pada konteks materi yang diberikan [14].

### 3.1.1. Sumber Data Primer: Korpus Materi Maguru

Sumber data utama penelitian ini adalah materi kursus **"Memulai Pemrograman dengan Python"** dari platform Maguru. Korpus ini terdiri dari 11 modul terstruktur yang mencakup 55+ unit pembelajaran (*lessons*) dalam format Markdown. Pemilihan format Markdown memfasilitasi proses ekstraksi informasi otomatis di mana elemen struktural seperti *headings* (`#`, `##`) digunakan sebagai penanda batas topik, dan *code blocks* diidentifikasi untuk dipertahankan integritas sintaksisnya selama proses *chunking*.

| ID Modul | Topik Utama | Cakupan Konsep |
| :--- | :--- | :--- |
| M01-M03 | Dasar Pemrograman | Variabel, Tipe Data, Ekspresi, Operator |
| M04-M05 | Kontrol Alur | Aksi Sekuensial, Percabangan, Perulangan (*Loops*) |
| M06-M07 | Struktur Data | Array, List, Matriks, Manipulasi Sekuens |
| M08-M11 | Konsep Lanjutan | Subprogram, OOP, *Style Guide*, *Unit Testing* |

### 3.1.2. Tahap 1: Domain Adaptation (Self-Supervised Learning)

Tahap pertama bertujuan untuk melakukan adaptasi domain (*domain adaptation*) agar model terbiasa dengan terminologi teknis dan gaya bahasa instruksional dalam bahasa Indonesia [10]. Proses ini menggunakan teknik pembelajaran tanpa pengawasan (*self-supervised*) pada korpus materi Maguru melalui dua format:
1.  **Span Corruption**: Mengikuti objektif pra-pelatihan asli T5, model dilatih untuk merekonstruksi bagian teks yang hilang (*masked spans*) [1]. Sebanyak 15% dari total token dalam materi di-*mask* menggunakan *sentinel tokens* (`<extra_id_n>`). Teknik ini terbukti efektif dalam meningkatkan pemahaman kontekstual model terhadap struktur kalimat teknis [7].
2.  **QA Generik (Rule-Based)**: Pasangan tanya-jawab sederhana diekstrak secara otomatis menggunakan heuristik berbasis aturan. Istilah kunci yang ditandai dengan format tebal (`**term**`) atau kode (`code`) dipetakan menjadi pertanyaan definisi. Tahap ini memberikan fondasi pengetahuan faktual tanpa memerlukan biaya komputasi model bahasa besar (LLM) eksternal.

### 3.1.3. Tahap 2: Task-Specific AQG (Instruction Tuning)

Tahap kedua berfokus pada pelatihan model untuk tugas spesifik pembuatan soal kuis pilihan ganda (MCQ). Dataset ini dibangun dengan prinsip **Context Grounding**, di mana setiap soal wajib bersumber dari informasi yang tersedia dalam teks konteks untuk menghindari bias pengetahuan eksternal model [15].

Dataset ini dihasilkan secara sintetis melalui bantuan LLM (GPT-4o) dengan instruksi yang ketat (*instruction tuning*) [4]. Setiap entri data terdiri dari:
-   **Input**: Gabungan antara konteks materi (250-400 token) dan instruksi tugas yang mencakup parameter topik, tingkat kesulitan, dan tipe soal.
-   **Target**: Sebuah string tunggal terstruktur yang berisi pertanyaan, jawaban benar, dan 3-4 distraktor yang dirancang berdasarkan miskonsepsi umum siswa (*misconception tags*).

Pendekatan *joint generation* pada tahap ini memungkinkan model IndoT5 untuk mempelajari korelasi antara konten materi dan pola distraktor yang masuk akal (*plausible distractors*), yang merupakan aspek paling menantang dalam pembuatan kuis otomatis [6].

3.2. Pra-pemrosesan & Rekayasa Prompt

Setelah data diakuisisi, tahap selanjutnya adalah mempersiapkan teks agar dapat diproses secara optimal oleh model IndoT5 melalui pipeline modular.

3.2.1. Modular Chunking dan Pembersihan Teks

Data mentah dari file Markdown diproses oleh komponen Chunker yang memotong teks menjadi potongan (chunks) berukuran 250–400 token berdasarkan batas heading. Proses ini memastikan keutuhan code block tetap terjaga. Pembersihan teks dilakukan untuk menghapus karakter non-esensial dan markup yang dapat mengganggu pemahaman model, sambil tetap mempertahankan metadata penting seperti source_file dan section_heading.

3.2.2. Konstruksi Prompt (Prompt Engineering)

Input untuk model dibangun menggunakan template tetap yang menggabungkan konteks materi dengan instruksi tugas spesifik. Rekayasa prompt dilakukan dengan menyertakan parameter seperti concept, difficulty (easy, medium, hard), dan question_type (MCQ atau Code Completion). Pendekatan ini memungkinkan kontrol yang lebih presisi terhadap karakteristik soal yang dihasilkan, memastikan kesesuaian dengan kebutuhan pedagogis siswa tingkat menengah di Indonesia
